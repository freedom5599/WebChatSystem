# 需求分析

> 文档版本：v1.0
> 最后更新：2026-06-10

---

## 一、项目背景

随着互联网技术的快速发展，即时通讯（IM）已成为人们日常沟通的重要方式。本课程大作业旨在通过开发一个 Web 版的在线聊天系统，让同学们深入理解：

- B/S 架构应用的开发流程
- 实时通信技术（WebSocket）
- 数据库设计与持久化
- 前后端分离/混合开发模式
- 团队协作与版本管理

---

## 二、用户角色

本系统仅有一种用户角色：**普通用户**（已注册用户）。所有用户功能对等，没有管理员后台。

## 三、功能需求

### 3.1 用户模块

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| U-01 | 用户注册 | 用户通过用户名 + 密码注册账号，可选填昵称 | 高 |
| U-02 | 用户登录 | 用户名 + 密码登录，错误时给出提示 | 高 |
| U-03 | 用户登出 | 清空会话，跳转登录页 | 高 |
| U-04 | 查看个人信息 | 显示自己的头像、昵称、个性签名 | 高 |
| U-05 | 修改个人信息 | 可修改昵称、头像、个性签名 | 中 |
| U-06 | 搜索用户 | 按用户名精确搜索其他用户 | 中 |

**业务规则**：
- 用户名长度 3-50 位，唯一
- 密码长度 6-20 位，使用 BCrypt 加密存储
- 未登录用户访问受限页面自动跳转登录页

### 3.2 好友模块

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F-01 | 发送好友申请 | 输入验证信息（可选），提交给目标用户 | 高 |
| F-02 | 收到好友申请 | 实时通知 + 待处理列表展示 | 高 |
| F-03 | 处理好友申请 | 同意或拒绝申请 | 高 |
| F-04 | 好友列表 | 显示所有好友（按分组归类） | 高 |
| F-05 | 好友分组 | 创建/重命名/删除好友分组 | 中 |
| F-06 | 移动好友 | 将好友从一个分组移动到另一个分组 | 中 |
| F-07 | 删除好友 | 从好友列表移除（双向） | 中 |
| F-08 | 好友备注 | 给好友设置备注名 | 低 |

**业务规则**：
- 不能添加自己为好友
- 已是好友不能重复申请
- 申请处理后记录从数据库删除（不留历史）

### 3.3 私聊模块

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| P-01 | 发送文字消息 | 在私聊窗口发送文本 | 高 |
| P-02 | 接收私聊消息 | WebSocket 实时接收 | 高 |
| P-03 | 查看历史消息 | 加载与某好友的历史消息 | 高 |
| P-04 | 发送语音消息 | 录制并发送语音 | 中 |
| P-05 | 发送图片消息 | 上传图片并发送 | 中 |
| P-06 | 发送文件消息 | 上传文件并发送 | 中 |
| P-07 | 标记已读 | 进入聊天窗口时标记对方消息已读 | 中 |
| P-08 | 未读消息数 | 显示未读消息数量 | 中 |
| P-09 | 导出聊天记录 | 导出为文本文件 | 低 |

### 3.4 群聊模块

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| G-01 | 创建群组 | 群主创建群，群成员至少 1 人（自己） | 高 |
| G-02 | 解散群组 | 群主可解散群（所有成员自动退出） | 高 |
| G-03 | 退出群组 | 普通成员可主动退出 | 高 |
| G-04 | 加入群组 | 通过群 ID 直接加入 | 中 |
| G-05 | 邀请入群 | 群成员邀请好友入群（需对方同意） | 中 |
| G-06 | 处理入群邀请 | 接受/拒绝群邀请 | 中 |
| G-07 | 踢出成员 | 群主踢人 | 中 |
| G-08 | 群成员列表 | 查看群所有成员 | 中 |
| G-09 | 群聊消息 | 发送/接收群消息 | 高 |
| G-10 | 群历史消息 | 查看群历史消息 | 中 |

**业务规则**：
- 群名称不能为空，长度 1-50 位
- 群主不能退出自己的群（需先解散或转让，群主转让功能本期未实现）
- 邀请入群后，记录在受邀人接受/拒绝时被删除

### 3.5 实时通讯模块

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| R-01 | 消息推送 | 私聊/群聊消息实时推送到目标 | 高 |
| R-02 | 在线状态 | 实时显示好友在线/离线 | 高 |
| R-03 | 通知推送 | 好友申请、群邀请等事件实时通知 | 中 |
| R-04 | 重连机制 | WebSocket 断线后自动重连 | 中 |

## 四、非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 并发用户数 | ≥ 100 |
| 消息延迟 | < 500ms（局域网） |
| 页面加载时间 | < 2s（首屏） |
| 接口响应时间 | < 300ms（常规接口） |

### 4.2 安全需求

| 项 | 说明 |
|----|------|
| 密码加密 | BCrypt 加密存储，不可逆 |
| 会话管理 | 基于 HttpSession，未登录拦截 |
| SQL 注入 | 使用 MyBatis 预编译参数 |
| XSS 防护 | 前端输入校验 + Thymeleaf 自动转义 |
| CSRF | 本期未实现（demo 性质） |

### 4.3 可用性需求

- 操作流程符合用户习惯（QQ/微信 风格）
- 错误信息友好提示
- 重要操作有确认弹窗
- 兼容性：Chrome 80+、Edge 80+、Firefox 80+

### 4.4 可维护性需求

- 代码分层清晰（Controller / Service / Mapper / Entity）
- 关键业务有注释
- 统一的返回格式 `Result<T>`
- 统一的异常处理

### 4.5 部署需求

- 支持 Windows / Linux 一键启动
- 数据库脚本可重复执行
- 提供可执行 jar 包

## 六、约束与限制

- 仅支持单服务器部署，不支持集群
- 不支持离线消息推送（需客户端在线）
- 不支持消息撤回、引用回复等高级功能
- 客户端仅限 Web 浏览器，不做移动端
- 不做管理员后台（无用户管理、封禁等功能）

## 七、验收标准

| 项 | 标准 |
|----|------|
| 功能完整 | 上述所有高优先级功能均可正常使用 |
| 实时性 | 私聊/群聊消息发送后 1 秒内对方可见 |
| 数据正确 | 刷新页面、重新登录后数据不丢失 |
| 健壮性 | 异常操作（重复登录、超长输入等）不导致系统崩溃 |
| 文档完整 | 需求、架构、API、部署、用户手册、测试报告齐全 |
